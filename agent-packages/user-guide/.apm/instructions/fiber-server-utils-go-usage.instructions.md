## Skill trigger: `fiber-server-utils-go-usage`

Use skill `fiber-server-utils-go-usage` when creating or modifying a Go microservice that exposes HTTP/REST endpoints, when wiring fiber.App, or when reviewing code that calls fiber.New / fiberserver.New / app.Listen / app.Get|Post|Put|Delete. Also use when adding health, prometheus, pprof, tracing, api-version, or TMF error handling to a fiber service. Do NOT use for non-fiber HTTP stacks (net/http, gin, echo, chi) or for pure gRPC/Kafka-only services.
